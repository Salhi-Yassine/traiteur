# Naming Conventions

## Translation Keys

**Pattern:** `<section>.<element>[.<qualifier>]` — always `snake_case`, always in the `common` namespace.

**Canonical section names** (derive new ones from the page or component name):

| Section | Used for |
|---|---|
| `nav` | Navigation items |
| `auth` | Login, register, password reset |
| `vendor_card` | VendorCard component strings |
| `vendor_profile` | Vendor detail page |
| `filters` | Filter sidebar and chips |
| `search_bar` | SearchBar component |
| `home` | Landing page |
| `planning` | Planning tools (budget, guests, checklist) |
| `errors` | Error messages and states |
| `common` | Reusable cross-component strings (save, cancel, next, etc.) |

**Element:** lowercase `snake_case` verb_noun or noun — e.g. `view_profile`, `starting_at`, `no_results`, `clear_all`.

**Qualifier** (only when needed for state/variant): `loading`, `error`, `empty`, `placeholder`, `title`, `subtitle`.

**Examples:**
```
vendor_card.view_profile       ✓
vendor_card.starting_at        ✓
common.saved                   ✓
common.currency                ✓
errors.something_went_wrong    ✓
auth.login_btn                 ✓
filters.clear                  ✓
```

**Forbidden:**
```
vendorCard.viewProfile         ✗  (camelCase)
title                          ✗  (too generic — always qualify: vendor_card.title)
label                          ✗  (too generic)
text                           ✗  (too generic)
VendorCard.view_profile        ✗  (PascalCase section)
```

**Adding a new string — always in this order:**
1. Choose the key following the pattern above
2. Add to `pwa/public/locales/fr/common.json` (primary — full text)
3. Add to `ar/common.json`, `ary/common.json`, `en/common.json` (placeholder = same as fr is acceptable; mark with a TODO comment in the PR if untranslated)
4. Use `t('your.key')` in code — never commit with a missing locale file entry

## TypeScript / React

| Thing | Convention | Example |
|---|---|---|
| Component file | `PascalCase.tsx` | `VendorCard.tsx` |
| Story file | `PascalCase.stories.tsx` co-located | `VendorCard.stories.tsx` |
| Page file | `kebab-case.tsx` (Next.js routing) | `[slug].tsx`, `index.tsx` |
| Props interface | `<ComponentName>Props` — exported | `export interface VendorCardProps` |
| API response shape | Match entity name | `VendorProfile`, `BudgetItem` |
| Hook file | `camelCase.ts` | `useVendorProfile.ts` |
| Utility file | `camelCase.ts` | `apiClient.ts` |

**Forbidden:**
- Barrel `index.ts` files inside `components/` — import directly from the file
- Inline type annotations replacing exported interfaces (e.g. `function Foo({ name }: { name: string })` for a public component)
- `any` for API response shapes — always define the interface

## Symfony / PHP

| Thing | Convention | Example |
|---|---|---|
| Entity class | Singular `PascalCase` | `VendorProfile`, `BudgetItem` |
| Serialization group (read) | `<entity-snake>:read` | `vendor:read`, `budget_item:read` |
| Serialization group (write) | `<entity-snake>:write` | `vendor:write`, `budget_item:write` |
| Voter attribute constant | `'<noun>:<action>'` | `'profile:edit'`, `'profile:create'` |
| Repository | `<Entity>Repository` | `VendorProfileRepository` |
| Service | `<Domain>Service` | `BudgetCalculatorService` |
| State Provider | `<Entity>Provider` | `MeProvider` |
| State Processor | `<Entity>Processor` | `VendorOnboardingProcessor` |
| Migration class | Generated by Doctrine — never rename | — |

**Serialization group examples from the codebase:**
```php
// VendorProfile — vendor:read / vendor:write
// (NOT VendorProfile:read — always use the snake-case entity short name)
#[Groups(['vendor:read', 'vendor:write'])]
```
